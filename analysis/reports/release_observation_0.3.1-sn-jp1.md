# 0.3.1-sn-jp1 发版观察清单

## 版本定位

`0.3.1-sn-jp1` 是第一轮维度校准实验版。

本版只验证一件事：

```text
仅通过 questionDimensionWeights.json 的维度权重覆盖，能否提升高置信 feedback 的一致率。
```

本版不验证：

- 题面重写效果
- 角色池重排效果
- 角色概率重算效果
- 自动逻辑回归全量权重效果

## 上线后必看指标

按 `app_version` 过滤：

```sql
SELECT app_version, COUNT(*)
FROM mbti_feedback
GROUP BY app_version
ORDER BY app_version;
```

核心指标：

- 高置信 feedback 数量
- 高置信完整四字母一致率
- `E/I` 一致率
- `S/N` 一致率
- `T/F` 一致率
- `J/P` 一致率
- 自报 MBTI 分布
- 预测 MBTI 分布
- 高频错配 pair

## 判定阈值

继续保留本版的条件：

- 高置信完整一致率高于 `0.3.0`
- `S/N >= 0.60`
- `J/P >= 0.61`
- `E/I >= 0.60`
- 高置信 feedback 数量没有明显下降

需要回滚或弱化的条件：

- `E/I < 0.58`
- 完整一致率没有提升
- `INFP / INTP / INFJ / ENFP` 等高频自报类型出现异常大规模错配
- 预测 MBTI 分布明显向少数类型塌缩

## 下一轮排查优先级

如果线上复现 `S/N、J/P 提升，但 E/I 下滑`：

1. 固定其他覆盖项。
2. 先弱化或移除 `q23` 覆盖。
3. 再单独复核 `q37` 覆盖。
4. 不继续增加新题覆盖。

重点怀疑项：

- `q23: {"S_N": -0.9}`
- `q37: {"S_N": -0.8, "J_P": 0.25}`

## 不做事项

- 不把 `src/data/questions.json` 的当前未提交大范围改动混入本版。
- 不把 `src/data/characters.json` 的 `matchWeight` 改动混入本版。
- 不把 `src/data/characterProbabilities.json` 的重算结果混入本版。
- 不提交 `scripts/*match_weights*.cjs` 或 `scripts/analyze_raw_scores.cjs`。
- 不在没有新版 feedback 的情况下继续叠第二轮维度权重。
